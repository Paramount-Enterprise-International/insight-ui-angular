Section

i-section & Related Components

The Section component system provides a structured layout container with header, body, footer, filter, and tabbed content areas. It's designed for organizing content into logical sections with consistent styling and behavior.


Components Overview

The Section system consists of these components:

  • i-section - Main container component
  • i-section-header - Section header area
  • i-section-sub-header - Section sub-header area
  • i-section-filter - Filter controls area
  • i-section-body - Main content area
  • i-section-footer - Footer area
  • i-section-tabs - Tabbed content container
  • i-section-tab - Individual tab
  • i-section-tab-header - Custom tab header template
  • i-section-tab-content - Custom tab content template

Component Properties

i-section-tab Properties

Property Type Required Default Value Description
title string Required '' Tab title displayed in the header
opened boolean Optional false If true, this tab will be active by default
badge any Optional null Badge indicator for the tab (see Badge Types below)

i-section-tabs Properties

Property Type Required Default Value Description
selectedIndex number | null Optional null Controlled mode - active tab index
selectedIndexChange EventEmitter<number> Optional - Emitted when active tab changes
height any Optional 'wrap' Tab content height (see Height Options below)

Event Properties

Property Type Required Description
onSelectedIndexChange EventEmitter<number> Optional Output event that emits the new selected tab index when tab selection changes

Height Options

The height property controls tab content height:

Value Result Example
null or "wrap" or "auto" Content height wraps to content height="wrap"
"300" or 300 Fixed 300px height with scroll height="300"
"500px" Fixed 500px height with scroll height="500px"
"80vh" Fixed 80vh height with scroll height="80vh"
