Card

i-card

The i-card component is a versatile container that can function as a clickable card, a navigation element, or a regular content wrapper. It supports three distinct interaction modes: traditional hyperlinks, Angular Router navigation, and click event handling.


Properties

Property Type Required Default Description
disabled boolean Optional false Disables all interactions and navigation. When true, sets appropriate ARIA attributes and prevents clicks.
href string | null Optional null Traditional anchor URL. Cannot be combined with routerLink.
routerLink string | any[] | undefined Optional undefined Angular Router navigation destination. Cannot be combined with href.
queryParams Record<string, any> | null Optional null Query parameters for router navigation.
fragment string Optional - URL fragment (hash) for navigation.
replaceUrl boolean Optional false Replaces browser history instead of adding new entry.
skipLocationChange boolean Optional false Navigates without changing browser URL.
state Record<string, any> Optional - Additional state data for router navigation.
target '_self' | '_blank' | '_parent' | '_top' | string Optional - Link target attribute. Auto-adds security attributes for _blank.
rel string | null Optional null Link relationship attribute.

Events

Event Type Description
onClick MouseEvent Emitted when card is clicked in click-only mode.

Child Components

i-card-image

Displays an image within the card. Requires src attribute.

i-card-body

Main content container. No additional properties required.

i-card-footer

Footer content container. No additional properties required.
